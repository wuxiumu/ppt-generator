### 叙事结构分析

Planner 自动识别为 **pitch（说服提案）** 叙事类型：

- Act 1（开场 4 张）：封面 → 痛点场景（"上个月大促，订单系统又崩了"）→ 数据冲击（big_number: "今年第4次"）→ 路线图
- Act 2（主体 19 张）：4个子章节——
  - 现状分析：Python 系统的性能瓶颈 → 大促超时数据 → 业务损失量化
  - 为什么是 Rust：语言对比（comparison: Python vs Rust）→ 性能数据 → 安全性优势 → 生态成熟度
  - 实施计划：分阶段迁移方案 → 时间表 → 人员配置 → 风险评估
  - ROI 分析：开发成本 vs 运维节省 → 大数字（"预计节省 ¥XX万/年"）→ 竞品案例
- Act 3（收尾 4 张）：行动建议 → 时间窗口（"Q3启动最合适"）→ 金句 → 结束

### 调色板选择逻辑

`商务提案` + `说服CEO` → Planner 自动选择 `corporate` 调色板（商务蓝 #2B6CB0 + 活力橙 #ED8936 + 成功绿 #38A169），专业且有说服力，适合向决策者汇报。

### 受众适配

`audience: "非技术 CEO"` 这个约束被 Planner 很好地利用：
- 避免技术术语，用"外卖比喻"解释并发问题
- 大量使用 big_number 布局展示业务数据（损失金额、超时次数）
- comparison 布局用的是"用户体验对比"而非"技术指标对比"
- 实施计划部分用 bullets 而非 code 布局
