## 02 - 叙事结构设计：如何保证"一套"而非"一堆"

### 核心思路：三幕叙事 + 叙事角色分配

把25-30张PPT当作一个完整的演讲来设计，借鉴电影编剧的三幕结构：

**Act 1 - 开场（4-5张，约15%）**：建立共鸣、抛出问题、预告方向
**Act 2 - 主体（17-20张，约65%）**：核心内容展开，分3-4个子章节
**Act 3 - 收尾（4-5张，约15%）**：总结升华、行动号召、情感落点

### 叙事弧线模板

```
Act 1: 开场
  ├─ S01  封面页：标题 + 副标题 + 视觉锤
  ├─ S02  钩子页：一个问题 / 一个反直觉的事实 / 一个场景
  ├─ S03  共鸣页："你是不是也遇到过..."
  ├─ S04  路线图：今天我们要聊什么（目录页）
  └─ S05  章节分隔：Act 2 开始
       │
Act 2: 主体（以"Python入门"为例）
  ├─ S06  子章节1标题：什么是编程
  ├─ S07  概念展开：编程 = 给电脑的指令
  ├─ S08  类比/举例：菜谱就是"程序"
  ├─ S09  章节分隔：进入子章节2
  ├─ S10  子章节2标题：变量——给数据起名字
  ├─ S11  概念展开 + 代码示例
  ├─ S12  动手试一试
  ├─ S13  章节分隔：进入子章节3
  ├─ S14  子章节3标题：循环——让电脑帮你重复
  ├─ S15  概念展开 + 生活类比
  ├─ S16  代码示例
  ├─ S17  小练习
  ├─ S18  章节分隔：进入子章节4
  ├─ S19  子章节4标题：函数——把步骤打包
  ├─ S20  概念展开
  ├─ S21  代码示例
  ├─ S22  综合小项目
  └─ S23  章节分隔：进入Act 3
       │
Act 3: 收尾
  ├─ S24  回顾页：今天学了什么（视觉化总结）
  ├─ S25  进阶路线图：下一步可以学什么
  ├─ S26  金句/激励页
  ├─ S27  资源推荐
  └─ S28  结束页：谢谢 + 联系方式
```

### Stage 1 Planner 的 Prompt 工程

Planner 是整个系统质量的瓶颈。它需要做到：

**1. 主题理解与受众适配**

不同受众的叙事策略完全不同：
- 零基础受众：从直觉出发，大量类比，降低门槛
- 同行受众：直接切入干货，少铺垫
- 决策者受众：先讲痛点/收益，后讲方案

**2. 子章节拆分**

将brief拆解为3-4个子章节，每个子章节5-6张slide：
```
子章节拆分规则：
- 每个子章节必须有独立的小叙事弧（引入→展开→小结）
- 子章节之间有逻辑递进关系（不是并列罗列）
- 难度递增：前简后难，前具体后抽象
```

**3. 布局类型分配**

为每张slide分配布局类型（详见03-视觉风格系统），确保视觉节奏变化：
```
布局节奏规则：
- 连续不超过2张纯文本slide
- 每3-4张出现一次视觉重的布局（全图/图表/对比）
- 章节分隔页必须是视觉上"留白多"的布局
- 封面和结束页的布局最特殊，形成首尾呼应
```

**4. 过渡设计**

每张slide的 content_brief 中必须包含：
- `from_context`：承接前一张的什么内容
- `to_context`：引向后一张的什么内容
- `transition_type`：递进 / 转折 / 举例 / 总结 / 对比

### 叙事连贯性校验（后处理）

Stage 2 生成完所有内容后，做一次后处理校验：

```python
def validate_narrative(slides):
    issues = []
    for i, slide in enumerate(slides):
        # 检查相邻slide的主题衔接
        if i > 0:
            prev_topic = slides[i-1].topic_keywords
            curr_topic = slide.topic_keywords
            if not has_overlap(prev_topic, curr_topic) and slide.layout != "divider":
                issues.append(f"S{i+1}: 与前一张缺少主题衔接")
        
        # 检查内容密度分布
        text_heavy_count = count_consecutive_text_heavy(slides, i)
        if text_heavy_count > 2:
            issues.append(f"S{i+1}: 连续{count}张纯文本，视觉疲劳")
    
    return issues
```

### 不同主题的叙事策略差异

| 主题类型 | 叙事策略 | 示例 |
|----------|----------|------|
| 教程/入门 | 渐进式：概念→示例→练习→综合 | Python入门 |
| 年度复盘 | 时间线式：Q1→Q2→Q3→Q4→展望 | 年度复盘 |
| 指南/攻略 | 决策树式：需求→选项→对比→推荐 | 咖啡豆挑选 |
| 说服/提案 | 问题-方案式：痛点→方案→证据→行动 | Rust重写 |
| 旅行攻略 | 行程式：Day1上午→下午→Day2→tips | 京都周末 |

Stage 1 Planner 根据 brief 和 audience 自动判断主题类型，选择对应的叙事模板。

### "钩子"设计

每张PPT的前3张（封面、钩子、共鸣）决定受众是否继续看。钩子页的设计策略：

- **反直觉数据**："全球每天产生 2.5 quintillion 字节的数据，但90%的人不知道什么是变量"
- **场景代入**："想象一下，你坐在咖啡馆，面前一杯刚磨好的手冲，但你完全不知道该点哪种豆子"
- **痛点共鸣**："上个月大促，订单系统又崩了。这是今年第4次了。"
- **直接提问**："如果周末只有两天，预算3000块，你怎么玩转京都？"

### Act 3 收尾的"升华"策略

结尾不能只是"谢谢观看"。好的收尾要让人有收获感：

- **回环式**：回到Act 1提出的那个问题/场景，但这次受众已经有答案了
- **行动号召**：给一个具体的、可立即执行的下一步
- **金句收束**：一句高度概括的话，让受众记住核心信息
- **彩蛋**：一个小惊喜（资源清单、隐藏推荐、二维码）
