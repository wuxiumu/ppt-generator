## 04 - 模型选型与成本控制

### 模型分级策略

不同阶段用不同级别的模型，这是成本控制的核心：

```
Stage 1 - Planner（全局规划）
  → 用最强的模型，因为：
    - 只调用1次，token开销可控
    - 这一步的质量决定整套PPT的上限
    - 输出约3000-5000 tokens，输入约1000 tokens
  → 候选：Claude 3.5 Sonnet / GPT-4o

Stage 2 - Writer（逐张内容）
  → 用中等模型，因为：
    - 每张slide一次调用（25-30次），次数多
    - 任务明确（有详细的content_brief指导）
    - 输出约200-400 tokens/slide
  → 候选：GPT-4o-mini / Claude 3.5 Haiku

Stage 2.5 - Image（可选图片生成）
  → 美观极致版用DALL-E 3 / Flux
  → 平衡版不用AI生图，用Unsplash API免费图库

Stage 3 - Renderer（纯代码）
  → 无LLM调用
```

### 成本精算

#### 美观度极致版（目标 < $10/份）

```
Stage 1 - Planner (Claude 3.5 Sonnet):
  输入: ~2000 tokens × $3/M = $0.006
  输出: ~5000 tokens × $15/M = $0.075
  小计: ~$0.08

Stage 2 - Writer (GPT-4o-mini × 28张):
  输入: ~1500 tokens/slide × 28 = 42000 tokens × $0.15/M = $0.006
  输出: ~400 tokens/slide × 28 = 11200 tokens × $0.60/M = $0.007
  小计: ~$0.01

Stage 2.5 - 图片生成 (DALL-E 3 × 8-12张):
  标准质量: $0.04/张 × 10 = $0.40
  HD质量: $0.08/张 × 10 = $0.80
  小计: ~$0.40 - $0.80

总计 LLM + 图片: ~$0.50 - $0.90
```

**结论**：LLM成本极低，主要成本在图片生成。即使全用HD DALL-E 3，也远低于$10。
这意味着可以在以下方面投入更多成本来提升美观度：
- 用更强的模型（Claude Opus / GPT-4o）做Writer，成本增加约$0.50
- 生成更多图片（15-20张），成本约$1.20
- 多次调用Planner做方案比较和择优，成本增加约$0.20

**实际成本优化空间极大，瓶颈不在LLM费用而在美观度上限。**

#### 美观度-成本-速度平衡版（目标 < $1/份，< 5分钟）

```
Stage 1 - Planner (GPT-4o-mini):
  输入: ~2000 tokens × $0.15/M = $0.0003
  输出: ~5000 tokens × $0.60/M = $0.003
  小计: ~$0.003

Stage 2 - Writer (GPT-4o-mini × 28张):
  小计: ~$0.01

Stage 2.5 - Unsplash免费图片 (5-8张):
  小计: $0

总计: ~$0.01/份
```

### 速度优化策略

**并行化**：
- Stage 2 的28张slide可以并行生成（asyncio + semaphore控制并发）
- Stage 2.5 的图片生成也可以并行
- 限制并发数避免API限流（OpenAI: 60 RPM for tier 2）

**缓存**：
- Planner 输出的全局规划可以缓存
- 相同topic的slide内容可以缓存（但对多样性不利）

**流式处理**：
- 不等待所有slide生成完再渲染
- 每生成完一张就进入渲染队列
- 图片生成和渲染可以交错进行

```python
import asyncio

async def generate_all_slides(slide_plan, style_spec):
    semaphore = asyncio.Semaphore(5)  # 最多5个并行
    
    async def gen_one(slide):
        async with semaphore:
            return await call_writer_llm(slide, style_spec)
    
    tasks = [gen_one(s) for s in slide_plan]
    results = await asyncio.gather(*tasks)
    return results
```

### 关于DeepSeek的考量

如果面向国内市场或想进一步压缩成本，DeepSeek-V3 是一个有竞争力的选项：
- 价格极低（$0.27/$1.10 per M tokens）
- 中文内容质量不错
- 但叙事规划能力可能不如Claude Sonnet
- 建议：Planner仍用Claude/GPT-4o，Writer可用DeepSeek做中文内容
